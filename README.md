2 обработчика (написаны в Dev\Site\Handlers). Для них так же использовал функции, которые вынес отдельно в Dev\Site\Helpers. Обработчики подключены в init.php 
новым способом. Обработчики срабатывают при изменении любого инфоблока, либо при создании нового. После этого создается новый инфоблок с логами (если его еще не было) и делается логирование


Написал агента, который удаляет все логи, кроме 10 самых новых. (написан в Dev\Site\Agents) Агент добавляется автоматически при установке модуля, регистрацию агента прописал в index.php директории install
Агент отображается в админке, срабатывает раз в час, при срабатывании не удаляется.

Разобрал два способа работы агентов - на хитах и на cron. Не совсем понял, какой лучше подходит для этого агента с одной стороны, агент выполняется очень быстро и не требует больших ресурсов,
с другой - было четко обозначено, что агент должен выполняться раз в час, а при выполнении на хитах в моменты маленькой нагрузки агенты могут не срабатывать. На моем сервере реализовал вариант на cron
